# Project Skills

Local Claude Code Agent Skills for this project. Each `<name>/SKILL.md` has YAML
frontmatter (`name`, `description`) describing **when** to apply it, plus a
senior-level playbook. Claude Code discovers these at session start; invoke with
`/<name>` or rely on description-based triggering.

> Authored in-repo (no marketplace/registry was available to install rated
> third-party skills). They are real working playbooks, not stubs.

| Skill | Applies to |
|---|---|
| `ui-ux-design` | Building/reviewing any user-facing UI, layout, visuals, interaction, motion |
| `frontend-architecture` | Component boundaries, state/data flow, module layout, large UI changes |
| `accessibility` | Keyboard, screen-reader, contrast, semantics, motion — WCAG 2.1 AA |
| `performance-optimization` | Render/list/bundle/animation perf; measure-first |
| `debugging` | Reproduce → isolate → root-cause → fix → verify |
| `security-auditing` | Auth/RBAC, sensitive data, scope isolation, audit, web vulns |
| `automated-testing` | Unit/integration/E2E; behavior + edge cases; deterministic |
| `code-refactoring` | Structure without behavior change; small verified steps |
| `product-thinking` | What to build & why; cut vanity; honest data story |
| `deep-reasoning` | Hard/ambiguous/high-stakes decisions; decompose + self-verify |
